# 调试记录

## 1. 问题1：MNIST数据集下载失败
- **现象**：运行代码时，数据集下载进度卡住，报错 `Connection refused`。
- **原因分析**：国内网络环境下，PyTorch默认的MNIST下载源访问不稳定，且代码中使用了包含中文的路径，导致数据存储失败。
- **修改点**：
  1. 将数据集存储路径修改为纯英文路径，如 `./data/mnist`。
  2. 手动下载MNIST数据集文件，放入对应目录，或更换国内镜像源下载。

## 2. 问题2：GPU运行时报错CUDA out of memory
- **现象**：模型训练开始后，程序崩溃并报错 `CUDA out of memory`。
- **原因分析**：当前GPU显存不足，无法容纳设置的批量大小（64）的训练数据与模型参数。
- **修改点**：
  1. 减小批量大小至32或16，降低单批次数据占用的显存。
  2. 修改代码，强制使用CPU运行：`device = torch.device("cpu")`。

## 3. 问题3：模型测试准确率极低（<20%）
- **现象**：训练5轮后，测试集准确率仅为10%左右，与随机猜测结果相当。
- **原因分析**：数据预处理中未对MNIST图像进行归一化，输入数据分布与模型训练预期不符；同时检查发现卷积层输出尺寸计算错误，导致全连接层输入维度不匹配。
- **修改点**：
  1. 在数据预处理中加入归一化操作：`transforms.Normalize((0.1307,), (0.3081,))`。
  2. 重新核对卷积与池化层的输出尺寸，修正全连接层的输入维度。

## 4. 问题4：LeNet-5模型训练时损失不下降
- **现象**：LeNet-5训练过程中，损失值始终保持在较高水平，准确率无明显提升。
- **原因分析**：LeNet-5的最后一层卷积层输出维度与全连接层的输入维度不匹配，导致前向传播出错，梯度无法正常更新。
- **修改点**：
  1. 核对LeNet-5各层的输出尺寸，确认最后一层卷积层输出为120×1×1，展平后维度为120，与全连接层的输入维度一致。
  2. 调整优化器的初始学习率为0.001，避免学习率过高导致参数更新震荡。
